### 2.2 Real-bug discoveries surfaced through this cycle

Two production-relevant bugs were caught + fixed during the cycle, both demonstrating dogfood-discovery discipline that auditors should treat as load-bearing evidence of engineering rigor:

1. **Sprint 356 — `_SEL_FTNS` selector wrong on mainnet.** The original sprint-302 commit pinned `_SEL_FTNS = "0x9b03f021"` for the `ftns()` getter on RoyaltyDistributor v2; the correct keccak256("ftns()")[:4] is `0xefa21b41`. On real Base mainnet RPC, the backend's `eth_call` against a wrong selector returns `0x` → the harness mapped None → SKIPPED. INV-RD-4 — explicitly documented as "THE money invariant" in the registry — was silently skipping rather than catching insolvency drift. Mocked-backend unit tests passed because they indexed by whatever selector the registry held, verbatim. Bug only surfaced when extending the harness to additional contracts forced selector canonicalization. Same dogfood-discovery class as sprint 347 (consensus_submitter ReceiptLeaf v2).

2. **Sprint 363 — halmos caught a real proof bug during development.** Initial `check_revoke_role_admin_gated` asserted `hasRole(DEFAULT_ADMIN_ROLE, caller)` in POST-state. Halmos counterexample: `caller=0xdead, role=bytes32(0)` — caller granted admin via setUp's chain, then self-revoked their OWN role. Documented OZ AccessControl behavior (renounceRole / revokeRole(role, self)). Post-state correctly shows caller no longer admin, but proof intent was "caller HELD admin AT CALL TIME" — a pre-state check. Fixed by snapshotting `caller_was_admin` BEFORE the prank. Documented inline in the .t.sol header as audit-visible example of halmos forcing precise intent.

### 2.3 Why a single ratification doc

Each sprint's individual deliverables fall within founder's existing operational authority under the Foundation governance charter (engineering work, doc updates, infrastructure improvements). What requires explicit council ratification at end-of-cycle is:

1. **Coherence of the Vision §14 item 4 expansion.** Sprint 302 ratified 5 invariants on 1 contract; this cycle expanded to 20 invariants on 7 contracts (runtime) + 81 symbolic proofs across 13 specs (halmos) + CI gate + unified operator endpoints. Together this constitutes the deepest §14 item 4 expansion to date — closing all named honest-scope items in §7.34 and meeting the Vision §14 commitment "formal verification on the highest-value contracts" with a verifiable, audit-runnable surface.

2. **Source-identity CI parity gate as binding regression discipline** (R-2026-05-13-1 below). The gate, once active, has authority-of-CI semantics: any change-set that mutates a cited canonical-source range without coordinated spec update + pin regen will fail CI red. This is the engineering-discipline mechanism that prevents silent drift between halmos symbolic proofs and what they claim to mirror.

3. **Fleet kill-switch operator runbook design-only status anchor as binding regression discipline** (R-2026-05-13-2 below). The runbook explicitly anchors to date "2026-05-13" and contains the load-bearing claim that the fleet-coordination layer remains design-only. Any future revision flipping that status without a corresponding §7-promotion-trigger event + audit-prep §7.X entry would mislead operators + auditors. The pin-test mechanism in `tests/unit/test_fleet_kill_switch_runbook.py` enforces this.

4. **Two real bugs caught during the cycle** (§2.2 above) — placing them on the formal record as audit-visible evidence of the discovery discipline.

## 3. Resolved

**RESOLVED 1.** The 26-sprint cycle (sprints 355–380, 26 merge-ready tags all dated `20260513`) is ratified as a coherent shipping increment.

**RESOLVED 2.** The Vision §14 item 4 formal-verification surface — runtime probe (7 contracts / 20 invariants) + halmos symbolic-execution lane (13 specs / 81 proofs / 16 mirrored invariants + 4 off-chain subsystems) + unified `/admin/formal-verification/*` endpoint family + source-identity CI parity gate — is recognized as meeting the Vision §14 commitment in its current scope. Auditors evaluating the §14 item 4 mitigation against the documented "15-30% probability of material exploit incident in the first three years" can run the full surface independently via the `prsm_formal_verification` MCP tool, the `/admin/formal-verification/*` endpoints, and (with halmos+forge installed) `halmos --contract <name>` against any spec.

**RESOLVED 3.** The §7.29 honest-scope SPOF gap is closed via the multi-bootstrap fallback arc (sprints 375–377, 380). Libp2p-mode operators no longer ignore the EU+APAC fallback list scaffolded in NodeConfig. The active bootstrap URL is observable across REST, MCP, CLI, and Prometheus surfaces.

**RESOLVED 4. (R-2026-05-13-1 — source-identity CI parity rule).** Every halmos symbolic spec in `contracts/symbolic-proofs/test/*.t.sol` that cites canonical-source line ranges (via the `(prsm|contracts)/[path]:NNN(-NNN)?` format) MUST have its citations hash-pinned in `contracts/symbolic-proofs/source_identity_pins.json`. Any change-set that mutates the cited canonical-source range MUST also update the spec semantically AND regenerate the pin via `scripts/update_source_identity_pins.py` AND land all three in the same change-set with commit-message cross-reference. Auto-regen on CI is explicitly prohibited — pin updates carry the engineer's attestation that the spec still mirrors canonical source semantically. The CI gate `tests/unit/test_source_identity_parity.py::test_live_source_identity_parity` is the binding enforcement mechanism. Supersession of this rule requires a new CR amending or revoking it, with the same change-set updating the gate accordingly.

**RESOLVED 5. (R-2026-05-13-2 — fleet kill-switch runbook anchor rule).** The fleet kill-switch operator runbook at `docs/operations/fleet-kill-switch-operator-runbook.md` contains the load-bearing anchor "design-only as of 2026-05-13" naming the current status of the fleet-coordination layer. Any future revision flipping that anchor to "shipped" or removing it MUST be accompanied by: (a) a documented §7 promotion-trigger event (T1–T6 per FLEET-KILL-SWITCH-SCOPING-1) with audit-trail evidence, (b) a follow-on audit-prep §7.X entry documenting the as-shipped state, and (c) a new CR superseding this anchor. The CI pin `tests/unit/test_fleet_kill_switch_runbook.py::test_runbook_does_not_claim_fleet_layer_is_live` is the binding enforcement mechanism. Supersession follows the same protocol as R-2026-05-13-1.

**RESOLVED 6.** The two bugs documented in §2.2 (`_SEL_FTNS` selector + halmos proof self-revoke counterexample) are formally on the record as dogfood-discovery audit-visible evidence. The fixes shipped in sprint 356 and sprint 363 respectively; this resolution is the institutional acknowledgment.

**RESOLVED 7.** The following trigger-gated paths remain pre-committed but NOT pre-authorized:

- Fleet kill-switch Phase F1 implementation — gates on §7 triggers (T1–T6). PRSM-CR-2026-05-08 Resolution 7 + FLEET-KILL-SWITCH-SCOPING-1 §6.1 bind the authority schedule. This resolution does NOT modify or supersede.
- Halmos Certora integration (alternative symbolic engine named in `prsm/economy/web3/formal_invariants.py` module docstring) — out of scope for this cycle. Engagement gates on PRSM-POL-2 external-audit-trigger conditions.
- Direct symbolic proofs against canonical contracts (vs structural-mirror specs) — requires modeling IERC20 + IProvenanceRegistry as symbolic backing contracts. Deferred; structural-mirror specs with source-identity pinning are the current canonical authority.

## 4. Conditions

1. **Anchor preservation.** R-2026-05-13-1 + R-2026-05-13-2 are binding council-commitment-level regression rules. The CI gates encoded in `tests/unit/test_source_identity_parity.py` + `tests/unit/test_fleet_kill_switch_runbook.py` are the enforcement mechanism. Removing or weakening either gate without a superseding CR is a P0 audit finding.

2. **Bug-discovery discipline.** The dogfood-discovery pattern that surfaced the `_SEL_FTNS` and self-revoke bugs in this cycle is the load-bearing engineering culture this CR formalizes. Future engineering work that introduces classes of mocked-test-passes-but-mainnet-fails bugs without a corresponding live-verification step is a class of regression this resolution flags for audit attention.

## 5. Explicit non-scope

This resolution does NOT:

1. Authorize the fleet-coordination layer's Phase F1 implementation. R-2026-05-13-2 anchors the design-only status; lifting it requires a new CR per the protocol in §3 RESOLVED 5.
2. Modify the §7 promotion triggers from FLEET-KILL-SWITCH-SCOPING-1 §6.1. T1–T6 remain the canonical authority for trigger-fire decisions.
3. Authorize the Coinbase CDP commissioning or any execute path on `coinbase_offramp_initiate`. PRSM-CR-2026-05-08 R-2026-05-08-1 remains in force; that CR's commission ratification requires its own follow-on.
4. Authorize the Aerodrome pool-seed mainnet ceremony. PRSM-CR-2026-05-13-1's two remaining operative-condition rows (Prismatica USDC wire + pre-flight balance snapshot) remain pending; this resolution does NOT supersede or accelerate.
5. Engage external auditors. PRSM-POL-2 trigger conditions remain canonical for that decision.
6. Commission additional bootstrap hosts (EU/APAC). The Libp2pDiscovery code path consumes the fallback list, but the actual EU+APAC droplet provisioning + DNS records remain operator/governance-driven, not engineering-driven. The `DEFAULT_BOOTSTRAP_NODES` config in NodeConfig is aspirational until those hosts exist.
7. Expand the source-identity citation parser beyond the `(prsm|contracts)/[path]:NNN(-NNN)?` format. Future expansion to bare-name citations or different prefixes is a follow-on; for this cycle, the rooted-path convention is the binding contract.
