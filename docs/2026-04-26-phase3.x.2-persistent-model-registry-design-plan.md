## 1. Context & Goals

Phase 3.x.1 shipped a real `TensorParallelInferenceExecutor` whose constructor accepts a `Dict[str, ShardedModel]` model registry. The merge-ready tag (`phase3.x.1-merge-ready-20260426`) explicitly flags this as a production caveat:

> "In-memory model registry only (no signed model manifest yet) — registry persistence + tamper-evident model loading is deferred to a later phase."

That deferral is now this phase. The gap is concrete: a node operator restarting the executor process loses its model registry; nothing prevents a malicious operator from swapping shard bytes between the registry handing them to the executor and the inference call; nothing lets a remote verifier confirm "the model claimed in this receipt is the model whose weights actually ran." Closing all three is one small phase, not a multi-week effort.

### 1.1 Why now

The 3.x.1 merge-ready tag is fresh; closing this caveat in the same week keeps the surface coherent for any partner reviewing the MCP product. Waiting risks:
- **Caveat fatigue** — a "v1 caveat list" with one item is honest; with five items it reads as unfinished.
- **Consumer entrenchment** — every new caller that constructs the executor with a raw dict makes the eventual migration harder.
- **Verifier-trust gap** — the inference receipt already carries `model_id`, but receipts can't be cryptographically tied to any specific shard contents until a signed manifest exists.

### 1.2 Non-goals for Phase 3.x.2

- **Not DHT distribution of manifests.** Filesystem-backed only for v1; DHT is a separate scoping doc.
- **Not on-chain anchoring of manifest CIDs.** Anchor contract is a separate Phase 3.x.3 or Phase 4+ item.
- **Not stake-gated publishing.** Open-publish + signature-verify is the v1 trust model; reputation/stake gating is discovery-layer work.
- **Not model-revocation flow.** Revocation needs an authoritative registry source-of-truth, which we don't have until the on-chain anchor lands.
- **Not encrypted-manifest support.** Manifests are public metadata (the model_id is already public); encrypted manifests are a privacy-tier extension that doesn't belong in v1.
- **Not a hard-flip of `TensorParallelInferenceExecutor.__init__`.** Dict acceptance preserved via wrapper; deprecation lands in a later phase.

### 1.3 Backwards compatibility

- `TensorParallelInferenceExecutor(model_registry={...dict...})` keeps working — wrapped transparently into `InMemoryModelRegistry`.
- All Phase 3.x.1 unit + integration tests continue to pass without modification.
- `MockInferenceExecutor` is unchanged (it owns its own internal model list, not a registry).

### 1.4 Trust model

PRSM's existing receipt-verification posture: client verifies cryptographically; PRSM doesn't need to be trusted. The registry extends this to model loading:

| Concern | Defense |
|---|---|
| Operator swaps shard bytes between registry storage and executor input | `executor.execute()` calls `registry.get(model_id)` which verifies every shard's sha256 against the manifest before returning |
| Operator swaps the entire model | Manifest signature under publisher's key; client can pin `expected_publisher_node_id` per `model_id` |
| Manifest tampering at rest | Ed25519 signature over canonical bytes excludes the signature itself; tampering breaks verification |
| Replay of a stale manifest | `published_at` timestamp + future on-chain anchor (deferred); v1 trusts the publisher's monotonic versioning |
| Two manifests with the same model_id | First-write-wins on each node's filesystem; conflict surfaces at registration with a clear error |

## 3. Architecture

```
┌─────────────────────────────────────────────────────────────────┐
│ TensorParallelInferenceExecutor                                 │
│   ├─ self._registry: ModelRegistry  (was: Dict[str, ShardedModel])
│   ├─ register_model(model)        → self._registry.register(...) │
│   ├─ supported_models()           → self._registry.list_models()  │
│   └─ execute(request)                                              │
│        sharded = self._registry.get(model_id)  ← verifies on read │
└─────────────────────────────────────────────────────────────────┘
                          ▲
                          │
┌─────────────────────────┴───────────────────────────────────────┐
│ ModelRegistry (ABC)                                              │
│   register(model: ShardedModel, *, identity: NodeIdentity)       │
│       → ModelManifest                                            │
│   get(model_id: str) → ShardedModel                              │
│       (raises ModelNotFoundError or ManifestVerificationError)   │
│   list_models() → list[str]                                      │
│   verify(model_id: str) → bool   (no-side-effects audit)         │
│   get_manifest(model_id: str) → ModelManifest                    │
└──────────────────────────────────────────────────────────────────┘
       △                                       △
       │                                       │
┌──────┴───────────┐                  ┌────────┴────────────────┐
│ InMemoryModelReg │                  │ FilesystemModelRegistry │
│ - dict-backed    │                  │ - root: Path             │
│ - tests + scaffold│                 │ - root/<model_id>/       │
│ - no persistence │                  │     manifest.json        │
│                  │                  │     shards/              │
│                  │                  │       <shard_id>.bin     │
│                  │                  │ - on register: write +   │
│                  │                  │   sign manifest          │
│                  │                  │ - on get: read + verify  │
└──────────────────┘                  └──────────────────────────┘
```

### 3.1 ModelManifest format

```python
@dataclass(frozen=True)
class ManifestShardEntry:
    shard_id: str
    shard_index: int
    tensor_shape: tuple[int, ...]
    sha256: str            # hex digest of tensor_data bytes
    size_bytes: int

@dataclass(frozen=True)
class ModelManifest:
    model_id: str
    model_name: str
    publisher_node_id: str
    total_shards: int
    shards: tuple[ManifestShardEntry, ...]   # ordered by shard_index
    published_at: float                       # unix timestamp
    schema_version: int = 1
    publisher_signature: bytes = b""          # excluded from signing payload
```

**Canonical signing payload** (excludes `publisher_signature` itself):
```
b"prsm-model-manifest:v1|"
+ model_id + "|"
+ model_name + "|"
+ publisher_node_id + "|"
+ str(total_shards) + "|"
+ str(published_at) + "|"
+ "|".join(f"{e.shard_index}:{e.sha256}:{e.size_bytes}" for e in shards)
```

Same construction pattern as `InferenceReceipt.signing_payload()` (Phase 3.x.1 Task 2). Tensor shape is **not** in the signing payload — the sha256 of tensor_data already commits to a specific byte sequence; the shape is recoverable but not verification-load-bearing.

### 3.2 Filesystem layout

```
<root>/
├── <model_id>/
│   ├── manifest.json              # signed ModelManifest
│   └── shards/
│       ├── <shard_id_0>.bin       # raw tensor_data bytes
│       ├── <shard_id_1>.bin
│       └── ...
└── <other_model_id>/
    └── ...
```

- **manifest.json** is the source of truth; shards/*.bin are content-addressed by the manifest's sha256 entries.
- Filesystem reads verify each shard's sha256 against the manifest entry on `get()`. A flipped bit raises `ManifestVerificationError` — fail closed.
- No locking (single-writer assumption per node); concurrent registration of the same `model_id` raises `ModelAlreadyRegisteredError`.

### 3.3 Registry interface contract

```python
class ModelRegistry(abc.ABC):
    @abc.abstractmethod
    def register(self, model: ShardedModel, *, identity: NodeIdentity) -> ModelManifest:
        """Register and sign. Raises ModelAlreadyRegisteredError on duplicate."""

    @abc.abstractmethod
    def get(self, model_id: str) -> ShardedModel:
        """Load + verify. Raises ModelNotFoundError or ManifestVerificationError."""

    @abc.abstractmethod
    def list_models(self) -> list[str]: ...

    @abc.abstractmethod
    def get_manifest(self, model_id: str) -> ModelManifest: ...

    def verify(self, model_id: str) -> bool:
        """Audit-only (no plaintext returned). Default impl: try get(); catch."""
        try:
            self.get(model_id); return True
        except (ModelNotFoundError, ManifestVerificationError):
            return False
```

## 6. Risk register

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Filesystem layout incompatible with future DHT distribution | Medium | Medium | Manifest format is the wire format; filesystem is just one transport. DHT track will treat `manifest.json` bytes as the unit of distribution. |
| Performance: per-`get()` sha256 of every shard is expensive for large models | Medium | Low | Cache verified manifests in memory after first verify; invalidate on filesystem mtime change. Defer to Task 4 if benchmarks show a problem; not in scope otherwise. |
| Dict-back-compat path leaks scaffold-identity into receipts | Low | Low | The scaffold identity only signs the wrapper manifest, never the inference receipt; receipts continue to be signed by the node identity at the API layer. Document explicitly. |
| Two nodes register the same `model_id` independently | Medium | Medium | First-write-wins per node. Cross-node conflict resolution is a discovery-layer + on-chain-anchor problem, deferred. |
| Shard files grow unbounded across registrations | Low | Low | Out-of-scope: GC / pruning is a v2 concern. v1 documents that operators are responsible for disk hygiene. |
