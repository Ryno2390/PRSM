# Phase 3.x.8.1 — Streaming HTTP Endpoint — Design Plan

**Status:** draft
**Date:** 2026-04-28
**Author:** PRSM core
**Predecessors:** Phase 3.x.1 Task 5 (`POST /compute/inference` unary endpoint), Phase 3.x.6 (`ParallaxScheduledExecutor.execute()`), Phase 3.x.7 (`RpcChainExecutor`), Phase 3.x.8 (streaming wire path + MCP adapter — `phase3.x.8-merge-ready-20260428` tag at `391b92b0`).

---

## 1. Why this point release exists

Phase 3.x.8 shipped the streaming-token wire path, executor generator API, and MCP adapter — but the production HTTP surface is untouched. `mcp_server.handle_prsm_inference` still calls `_call_node_api("POST", "/compute/inference", ...)` which is unary. End users typing into the `prsm_inference` MCP tool see no UX change: 3.x.8 is dormant scaffolding until the HTTP layer surfaces it.

This point release closes that gap with a bounded scope change: add a sibling `POST /compute/inference/stream` endpoint that frames the executor's streaming output as Server-Sent Events (SSE); add a streaming HTTP client helper in the MCP server; route streaming-capable MCP contexts to the new endpoint. Same escrow/auth/budget gates as the unary path; same signed receipt at end-of-stream.

## 2. Scope

### 2.1 In scope

- **`POST /compute/inference/stream` endpoint** on the node API (`prsm/node/api.py`).
  - Same input schema as `/compute/inference`.
  - Returns `text/event-stream` (SSE) — one event per `StreamToken`, plus a terminal `result` event carrying the signed `InferenceResult`.
  - Reuses the existing `node.inference_executor` (`ParallaxScheduledExecutor`).
- **`ParallaxScheduledExecutor.execute_streaming(request)`** — streaming sibling of `execute()`. Drives `chain_executor.execute_chain_streaming(...)` instead of `execute_chain(...)`. Yields `InferenceTokenEvent` per token + a final `InferenceResult` (success or failure). All existing pre-execute gates (catalog lookup, budget, pool gathering, anchor verify, tier filter, allocation, routing) reused.
- **SSE wire format** — JSON-encoded events with discriminated `event:` field:
  - `event: token` — `data: {"sequence_index": N, "text_delta": "...", "token_id": null|int, "finish_reason": null|"stop"|...}`
  - `event: result` — `data: { ...InferenceResult.to_dict()... }` (terminal)
  - `event: error` — `data: {"error": "..."}` (terminal — dispatched on pre-execute failure)
- **`_call_node_api_streaming` helper** in `prsm/mcp_server.py` — opens the SSE connection, parses events, forwards token events to `emit_progress`, returns the final `InferenceResult`-shaped dict to the handler.
- **`handle_prsm_inference` routing** — when `emit_progress is not None` (the MCP client supplied a `progressToken`), call the streaming endpoint; else call the unary endpoint (back-compat).
- **E2E test** — full HTTP roundtrip via FastAPI TestClient, verifies incremental events are emitted in order + final receipt verifies under the node identity.

### 2.2 Out of scope (defer)

- **Real autoregressive runner.** v1 still uses `SyntheticStreamingRunner`. Production wiring of a real autoregressive decoder is Phase 3.x.10.
- **Caller-initiated cancellation over HTTP.** SSE has no native client→server cancel signal beyond `connection.close()`. Phase 3.x.8 Task 6 documents this v1 limitation; `connection.close()` triggers the server's cleanup path (Python GeneratorExit propagates through Starlette → executor → server). No partial-receipt-on-cancel pathway in v1; tests verify clean cleanup.
- **WebSocket-based bidi streaming.** SSE is one-way (server→client). v1 is sufficient for token output. Future bidi (e.g., for client-side cancel signal) is a Phase 3.x.8.x revisit.
- **Streaming + chunked-input composition.** Phase 3.x.8 v1 constraint preserved: `_dispatch_streaming_tail` rejects activations exceeding `chunk_threshold_bytes` with `ACTIVATION_TOO_LARGE`. End-to-end this means very long prompts can't currently stream; deferred to Phase 3.x.10.

### 2.3 Non-goals

- Replacing the unary `/compute/inference` endpoint. Streaming is additive — clients without streaming capability continue to use the unary path.
- New auth/escrow primitives. The streaming endpoint reuses the existing pre-execute gates verbatim (escrow lock → execute → release/refund on terminal `result` or `error`).
- Custom SSE framing. Standard `text/event-stream` per W3C SSE spec — `event:`/`data:` lines, blank-line frame separator.

### 3.2 Wire format — SSE event stream

Standard W3C SSE — easier than custom framing because every browser + every HTTP client library supports it natively. Per-event format:

```
event: token
data: {"sequence_index": 0, "text_delta": "Hello", "token_id": null, "finish_reason": null}

event: token
data: {"sequence_index": 1, "text_delta": " world", "token_id": null, "finish_reason": "stop"}

event: result
data: {"success": true, "request_id": "req-1", "output": "Hello world", "receipt": {"job_id": ..., "settler_signature": "...", "streamed_output": true, ...}, "cost_ftns": "0.5"}
```

On pre-execute failure (e.g., insufficient budget):
```
event: error
data: {"error": "Insufficient budget: 0.001 FTNS < required 0.5 FTNS", "code": "BUDGET_INSUFFICIENT"}
```

Properties:
- `event: token` events are NEVER terminal. Consumer keeps reading.
- `event: result` events are ALWAYS terminal. Consumer parses + closes connection.
- `event: error` events are ALWAYS terminal. Consumer raises a structured failure.
- A connection close BEFORE any terminal event is the same as cancellation. Server-side cleanup runs (Phase 3.x.8 Task 6 mechanism).

### 3.3 ParallaxScheduledExecutor.execute_streaming

```python
@dataclass(frozen=True)
class InferenceTokenEvent:
    """Per-token event yielded by execute_streaming. Mirrors
    StreamToken's user-visible fields without exposing the internal
    chain_rpc-layer wire types."""
    sequence_index: int
    text_delta: str
    token_id: Optional[int] = None
    finish_reason: Optional[str] = None


class ParallaxScheduledExecutor:
    # ... existing execute() unchanged ...

    async def execute_streaming(
        self, request: InferenceRequest,
    ) -> AsyncIterator[Union[InferenceTokenEvent, InferenceResult]]:
        """Streaming counterpart of execute(). Yields per-token events
        followed by exactly one terminal InferenceResult (success or
        failure).

        Pre-execute gates (catalog/budget/pool/anchor/tier/allocation/
        routing) reuse execute()'s exact code path. On pre-execute
        failure: yields a single InferenceResult.failure(...) and
        terminates — NO token events emitted.

        On success: yields N InferenceTokenEvents followed by one
        InferenceResult.success(...) carrying the signed receipt with
        streamed_output=True.
        """
```

Implementation:
1. Steps 1-6 (catalog → budget → pool → anchor → tier → allocation → route) reuse `execute()`'s logic — extract the shared `_pre_execute_gates(request) -> Union[GateResult, InferenceResult]` helper that returns either the chain to execute OR a structured failure.
2. If gate result is failure: yield it as the sole terminal event.
3. Otherwise, drive `self._chain_executor.execute_chain_streaming(request=request, chain=chain)`:
   ```python
   for item in self._chain_executor.execute_chain_streaming(...):
       if isinstance(item, StreamToken):
           yield InferenceTokenEvent(
               sequence_index=item.sequence_index,
               text_delta=item.text_delta,
               token_id=item.token_id,
               finish_reason=item.finish_reason,
           )
       elif isinstance(item, ChainExecutionResult):
           outcome = item
           break
   ```
4. Build + sign receipt via `_build_signed_receipt(streamed=True)` (the L3 round-1 follow-up — extract the streamed flag as a parameter so we don't duplicate the receipt-build code path).
5. Yield `InferenceResult.success(...)` with the signed receipt as the terminal event.

### 3.4 HTTP endpoint

```python
@app.post("/compute/inference/stream")
async def compute_inference_stream(body: Dict[str, Any] = {}):
    """SSE-streaming sibling of /compute/inference. Same input schema;
    response is text/event-stream with token + result events."""
    # ... validate body, build InferenceRequest, lock escrow ...
    # (identical to /compute/inference up through escrow lock)

    async def event_generator():
        try:
            async for item in node.inference_executor.execute_streaming(request):
                if isinstance(item, InferenceTokenEvent):
                    yield _sse_event("token", _json(_event_to_dict(item)))
                else:  # InferenceResult terminal
                    if item.success:
                        # Settle escrow on success.
                        escrow_settle(escrow_entry, item.receipt.cost_ftns)
                        yield _sse_event("result", _json(_result_to_dict(item)))
                    else:
                        # Refund escrow on pre-execute failure.
                        escrow_refund(escrow_entry)
                        yield _sse_event("error", _json({
                            "error": item.error,
                            "code": item.error_code or "EXECUTION_FAILURE",
                        }))
                    return
        except Exception as exc:  # noqa: BLE001
            # Defensive: any unhandled exception → SSE error event +
            # escrow refund. The executor is supposed to surface
            # failures as InferenceResult.failure, but a transport
            # crash mid-stream is recoverable here.
            escrow_refund(escrow_entry)
            yield _sse_event("error", _json({
                "error": f"{exc.__class__.__name__}: {exc}",
                "code": "INTERNAL_ERROR",
            }))

    return StreamingResponse(
        event_generator(),
        media_type="text/event-stream",
        headers={
            "Cache-Control": "no-cache",
            "X-Accel-Buffering": "no",  # disable nginx buffering
        },
    )


def _sse_event(event_type: str, data: str) -> bytes:
    """Encode a single SSE frame."""
    return f"event: {event_type}\ndata: {data}\n\n".encode("utf-8")
```

Escrow semantics:
- **Pre-execute failure** (budget gate, missing model, etc.) — escrow refunded, `event: error` emitted, no tokens.
- **Mid-stream failure** (rare — most failures surface at pre-execute) — escrow refunded if no tokens yet emitted; settled-as-billed if tokens emitted (caller paid for partial output). This is an OPEN DESIGN QUESTION — see §5 risk register.
- **Success** — escrow settled at the cost computed by `estimate_cost(request)`, same as unary path.

### 3.5 MCP client streaming consumer

```python
async def _call_node_api_streaming(
    path: str,
    data: Dict[str, Any],
    emit_progress: ProgressEmitter,
) -> Dict[str, Any]:
    """Open an SSE connection to a node-API endpoint, forward token
    events to emit_progress, return the final result dict.

    Raises ``RuntimeError`` if the stream terminates without a
    ``result`` event (server crashed, connection dropped, etc.).
    Raises ``InferenceError`` (subclass of RuntimeError) on
    ``error`` event."""
    async with httpx.AsyncClient(timeout=None) as client:
        async with client.stream(
            "POST", _node_url(path),
            json=data,
            headers={"Accept": "text/event-stream"},
        ) as response:
            response.raise_for_status()
            sequence = 0
            async for event_type, event_data in _parse_sse(response):
                if event_type == "token":
                    payload = json.loads(event_data)
                    sequence += 1
                    await emit_progress(
                        payload["text_delta"],
                        float(sequence),
                        None,
                    )
                elif event_type == "result":
                    return json.loads(event_data)
                elif event_type == "error":
                    err = json.loads(event_data)
                    raise InferenceError(err["error"], code=err.get("code"))
            raise RuntimeError(
                "SSE stream ended without a 'result' or 'error' event"
            )


async def _parse_sse(response) -> AsyncIterator[Tuple[str, str]]:
    """Minimal SSE parser. Yields (event_type, data) tuples."""
    event_type = "message"  # SSE default
    data_lines = []
    async for line in response.aiter_lines():
        if line == "":
            if data_lines:
                yield event_type, "\n".join(data_lines)
                data_lines = []
                event_type = "message"
        elif line.startswith("event: "):
            event_type = line[len("event: "):]
        elif line.startswith("data: "):
            data_lines.append(line[len("data: "):])
        # (comments + retry: lines ignored — minimal parser)
```

`handle_prsm_inference` modification:

## 5. Risk register

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Mid-stream escrow semantics are ambiguous (settle vs. refund on partial-output) | Medium | Low | v1 policy: pre-execute failure = refund; ANY token emitted = settle at full estimate (caller paid for chain dispatch). Document explicitly in endpoint docstring; revisit if operators report unfair-billing. Tracked as Phase 3.x.8.1.x billing-policy item. |
| SSE proxy/firewall buffering breaks real-time UX | Medium | Medium | `X-Accel-Buffering: no` + `Cache-Control: no-cache` headers set; documented operator guidance for nginx/cloudflare deployments. Alternative chunked-transfer encoding considered — SSE picked because every browser + every HTTP client supports it natively. |
| Connection-close mid-stream leaks server resources | Low | Medium | Phase 3.x.8 Task 6 cancellation cleanup propagates through Starlette's `StreamingResponse` close path → executor's GeneratorExit → server runner cleanup. Tested in Task 5 scenario (c). |
| `_call_node_api_streaming` parsing of malformed SSE frames | Low | Low | Minimal SSE parser ignores unknown lines (per spec); malformed `data:` JSON raises caught exception → `InferenceError`. Tested with malformed-frame fuzzing in Task 3. |
| HTTPX dependency — async client requirement | Low | Low | PRSM already depends on httpx for `_call_node_api`. The streaming variant uses the same client. No new deps. |
| Receipt's `streamed_output=True` flag downgrade attack mid-flight | Low | Low | Phase 3.x.8 Task 4 already commits the flag to the signed payload. A relay tampering the SSE `result` event would invalidate the receipt's signature, caught at MCP-side `verify_receipt` (post-tag follow-up — currently the MCP handler doesn't verify; tracked as Phase 3.x.8.1.x). |
| Concurrent streaming requests overload single node | Medium | Medium | v1: existing rate-limiting on the node API handles this at the connection level. SSE's per-connection lifetime + Starlette's async event loop handles backpressure naturally. Production tuning in Phase 3.x.10. |
