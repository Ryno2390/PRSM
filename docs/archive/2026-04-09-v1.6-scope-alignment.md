## Appendix A: Rollback Procedures

### If a PR introduces a regression not caught until later

1. Identify the PR commit SHA: `git log --oneline | grep <keyword>`
2. Create a revert branch: `git checkout -b revert/<pr-name>`
3. Revert: `git revert <sha>`
4. Run full test suite: `pytest tests/`
5. If tests pass, commit the revert and merge: `git checkout main && git merge revert/<pr-name>`
6. Re-plan the affected PR before attempting again

### If PyPI publish succeeds but smoke test fails

PyPI releases cannot be rescinded. Options:
1. Yank the release: `twine upload --skip-existing --verbose dist/*` does NOT yank; use `pip index versions prsm-network` then manually yank via pypi.org UI
2. Publish a v1.6.1 patch release with the fix
3. Never force-republish over an existing version

### If GitHub push fails

1. Check SSH key: `ssh -T git@github.com` — should greet by username
2. If key mismatch: do NOT modify git config; surface to the user
3. Retry push: `git push origin main` and `git push origin v1.6.0`
