## 1. Problem statement

### 1.1 Per-stage wire still leaks per-token timing

Phase 3.x.11.q's chain-level decorators (`BatchedTrailingShardedExecutor` M2, `FixedRateShardedExecutor` M1) wrap the chain-executor's `execute_chain_streaming` generator. They mask the executor ŌåÆ caller path: a wire observer with TLS termination on the executor's outgoing stream sees either one trailing frame (M2) or cadence-clamped frames (M1). The chain operates at native per-token speed underneath ŌĆö but the per-stage RPC dispatches (executor ŌåÆ alice ŌåÆ bob ŌåÆ ... ŌåÆ executor) continue at native cadence. A network observer with visibility on a single stage's transport learns the raw per-token decode rate, which (per ┬¦3.1) leaks tokens-per-second + token-id correlation under various analytical attacks.

The fix mirrors the chain-level pattern but applies AT EACH PER-STAGE BOUNDARY: each stage's outgoing dispatches are clamped to a configured cadence. Because per-stage timing is operator-controlled (the operator wires the M1/M2 decorator at server config time), this is a stage-deploy-time policy choice rather than a chain-wide one.

### 1.2 M2 response-size leaks total joined-text length

Phase 3.x.11.q's M2 (`BatchedTrailingShardedExecutor`) emits a single trailing `StreamToken` carrying the joined text. Wire observer sees the cumulative byte count of the response, which (under non-trivial token distributions) approximates the total token count. The ┬¦7.13 honest-scope item flagged this as deferred to q.x; under ┬¦3.8 q.y' the residual carried forward as item #4.

The fix is operator-configurable padding: M2 pads the joined-text up to a fixed maximum length (or pads to the next bucket in a configured size schedule, e.g. {64B, 256B, 1KiB, 4KiB}). Observer sees a constant-byte (or coarsely-bucketed) response regardless of actual content length.

## 2. Scope

In scope:

1. **Per-stage `BatchedTrailingLayerStageRunner` (M2)** ŌĆö wraps any object implementing the existing `StreamingLayerRunner` Protocol (Phase 3.x.10.y pattern). Drains the inner runner's full token stream, emits ONE final `StreamFinalFrame` with the joined text. Mirror of `BatchedTrailingStreamingRunner` (single-host, Phase 3.x.10.y) at the per-stage boundary.

2. **Per-stage `FixedRateLayerStageRunner` (M1)** ŌĆö clamps inter-token cadence on the outgoing per-stage wire to Ōēź `cadence_seconds`. Inter-token gap < cadence ŌåÆ sleep until cadence elapsed. Inter-token gap Ōēź cadence ŌåÆ emit immediately. Injectable clock + sleep for test determinism.

3. **`make_per_stage_constant_time_runner` factory** ŌĆö mode-string selection (`'m2'` / `'m1'`); cadence required for m1, rejected for m2; mirrors Phase 3.x.10.y + 3.x.11.q pattern.

4. **`LayerStageServer` integration** ŌĆö server constructor accepts the new optional `per_stage_constant_time_runner` kwarg (or operators wrap the existing `streaming_runner` at construction). Runtime invariant: `per_stage_constant_time_runner` ŌŖĢ `streaming_runner` are mutually exclusive (operator picks one or the other; the constant-time runner wraps the inner streaming runner).

5. **M2 response-size padding** ŌĆö `BatchedTrailingLayerStageRunner` constructor takes a `pad_to_bytes: Optional[int]` kwarg. When set, the joined-text payload is padded to exactly `pad_to_bytes` bytes (or, if joined text exceeds `pad_to_bytes`, the runner truncates with a `finish_reason="length_capped"` semantic ŌĆö same as the existing max_tokens cap path). Operator picks `pad_to_bytes` Ōēź expected max output length. Defends against the ┬¦7.13 honest-scope item #2 (M2 response-size leak).

6. **Composition with chain-level decorators (3.x.11.q)** ŌĆö M2 chain-level + M2 per-stage compose cleanly: chain-level emits one trailing frame, per-stage emits one trailing frame per stage. Operator wires both for full-network masking. M1 + M1 also composes (cadence chains; effective inter-frame interval at the caller = max(chain_cadence, per_stage_cadence ├Ś num_stages)).

7. **Threat-model + audit-prep updates** ŌĆö ┬¦3.7 rewrites the "per-stage wire still leaks" residual to closed (under operator opt-in); ┬¦3.8 honest-scope item #4 follows. New auditor reading-path entries for the per-stage runners + factory + server integration.

Out of scope (defer):

- **Adaptive cadence based on observed network load** ŌĆö Phase 3.x.11.q.z. Carries forward from ┬¦7.13.
- **Mid-stream cadence change** ŌĆö not supported (operator picks at deploy time; rotation requires redeploy).
- **Multi-stage replay forward** ŌĆö Phase 3.x.11.q.y'' if telemetry warrants.
- **Per-stage nonce cache** for replay-window-inside-deadline_unix defense ŌĆö orthogonal.
- **Padding schedule selection beyond fixed max** ŌĆö operator-configurable bucket schedule deferred (extension of pad_to_bytes; trivial to add later).

## 3. Design

### 3.1 Per-stage M2 (`BatchedTrailingLayerStageRunner`)

Mirrors `BatchedTrailingStreamingRunner` (single-host, Phase 3.x.10.y) at the per-stage boundary:

```python
class BatchedTrailingLayerStageRunner:
    def __init__(
        self,
        inner: StreamingLayerRunner,
        *,
        pad_to_bytes: Optional[int] = None,
    ) -> None:
        self._inner = inner
        self._pad_to_bytes = pad_to_bytes

    def stream_layer_slice(self, *, request, ...) -> Iterator[Union[TokenFrame, StreamFinalFrame]]:
        joined_text = []
        terminal: Optional[StreamFinalFrame] = None
        for frame in self._inner.stream_layer_slice(request=request, ...):
            if isinstance(frame, TokenFrame):
                joined_text.append(frame.text_delta or "")
            elif isinstance(frame, StreamFinalFrame):
                terminal = frame
                break
            else:
                # Pass through ŌĆö defer to inner's contract for unknown frame types.
                yield frame
        text = "".join(joined_text)
        # M2 padding: pad joined text to pad_to_bytes if configured.
        if self._pad_to_bytes is not None:
            current_len = len(text.encode("utf-8"))
            if current_len > self._pad_to_bytes:
                # Truncate at byte boundary, set finish_reason="length_capped".
                # Edge: don't split a multi-byte UTF-8 sequence.
                text = _safe_byte_truncate(text, self._pad_to_bytes)
                if terminal is not None:
                    terminal = dataclasses.replace(
                        terminal, finish_reason="length_capped",
                    )
            elif current_len < self._pad_to_bytes:
                # Pad with U+0020 (space). Reader is operator/MCP-aware
                # and strips trailing whitespace.
                text = text + " " * (self._pad_to_bytes - current_len)
        yield TokenFrame(
            sequence_index=0,
            text_delta=text,
            token_id=None,  # joined frame, no single token_id
            finish_reason=(
                terminal.finish_reason if terminal else None
            ),
        )
        if terminal is not None:
            yield terminal
```

**Padding correctness.** UTF-8 safe truncation requires walking back from `pad_to_bytes` to the last codepoint boundary; Python's `.encode("utf-8")[:n].decode(errors="ignore")` handles this with one caveat ŌĆö the encoded length may end up < pad_to_bytes if the truncation hit a multi-byte sequence. The runner re-pads with whitespace after truncation to hit the exact pad_to_bytes target. Round-trip: encoded bytes length is exactly pad_to_bytes regardless of input.

**Whitespace padding choice.** U+0020 (space) is the simplest. Trailing whitespace is convention-strippable by the MCP reader; a more robust choice is a non-printable sentinel (e.g. NUL bytes) but that would break MCP JSON-encoding. Whitespace is the reasonable default; operators who need a different padding character can override via constructor kwarg in a follow-up.

### 3.2 Per-stage M1 (`FixedRateLayerStageRunner`)

Mirrors `FixedRateStreamingRunner` (single-host, Phase 3.x.10.y):

```python
class FixedRateLayerStageRunner:
    def __init__(
        self,
        inner: StreamingLayerRunner,
        *,
        cadence_seconds: float,
        clock: Callable[[], float] = time.monotonic,
        sleep: Callable[[float], None] = time.sleep,
    ) -> None:
        self._inner = inner
        self._cadence_seconds = cadence_seconds
        self._clock = clock
        self._sleep = sleep

    def stream_layer_slice(self, *, request, ...) -> Iterator[...]:
        last_emit_ts = self._clock()
        for frame in self._inner.stream_layer_slice(request=request, ...):
            if isinstance(frame, TokenFrame):
                # Cadence clamp: sleep if needed.
                now = self._clock()
                gap = now - last_emit_ts
                if gap < self._cadence_seconds:
                    self._sleep(self._cadence_seconds - gap)
                last_emit_ts = self._clock()
                yield frame
            else:
                # Pass through; terminal frame doesn't get cadence-clamped.
                yield frame
```

**Cadence calibration is operator responsibility.** Mirrors ┬¦7.13 trust-seam item: cadence MUST be Ōēź chain native per-token rate at the stage; otherwise the runner yields immediately every tick and provides no masking. Recommended starting point: 2├Ś measured native rate at the stage.

### 3.3 Factory + LayerStageServer integration

`make_per_stage_constant_time_runner(inner, *, mode, cadence_seconds=None, pad_to_bytes=None)` mirrors the chain-level factory pattern. `LayerStageServer.__init__(streaming_runner=...)` already accepts a `StreamingLayerRunner`; operators just wrap their existing runner at server construction:

```python
inner = AutoregressiveStreamingRunner(...)
wrapped = make_per_stage_constant_time_runner(
    inner, mode="m2", pad_to_bytes=4096,
)
server = LayerStageServer(streaming_runner=wrapped, ...)
```

No new server kwarg needed. The wrap is composable with the existing Tier C dispatch-layer gate (Phase 3.x.10.y); the gate fires BEFORE the runner is invoked, so wrapping doesn't change tier-routing behavior.

### 3.4 Composition with Phase 3.x.11.q chain-level decorators

The chain-level decorator wraps `execute_chain_streaming` at the executor level; the per-stage runner wraps `stream_layer_slice` at the server level. They live on different sides of the wire and compose cleanly:

```
Caller ŌåÉŌöĆ chain-level M2/M1 ŌåÉŌöĆ executor ŌåÉŌöĆŌöĆŌöĆŌöĆ (per-stage cadence-clamped wire) ŌöĆŌöĆŌöĆŌöĆŌåÆ stage server ŌåÉŌöĆ per-stage M2/M1 ŌåÉŌöĆ runner
         (executor ŌåÆ caller)                                                                       (stage runner ŌåÆ executor)
```

Operators wire BOTH for full-network masking. Documented in the new ┬¦7.15 of audit-prep.

## 5. Open questions for the round-1 review

1. **M2 padding character.** Whitespace vs NUL vs operator-configurable? Whitespace is the v1 default; NUL would be cleaner but breaks JSON encoding. Defer operator-configurable until telemetry asks.
2. **M1 cadence calibration.** Auto-detect native rate at server startup (one-time profiling)? Or stay operator-responsibility per ┬¦7.13 pattern? v1 stays operator-responsibility ŌĆö auto-detect is a ┬¦7.13.q.z follow-up.
3. **Composition rules.** Chain-level M1 + per-stage M2 ŌĆö is that a sensible deploy? Probably no (M2 collapses to one frame; chain-level M1 has nothing to clamp). Document as discouraged but don't enforce; operators self-select.
4. **Per-stage cadence per-tier.** Should cadence depend on `request.content_tier`? Tier C requires aggressive cadence; Tier A might prefer no cadence at all. v1 ships single-cadence; tier-conditional cadence is a ┬¦7.13.q.z follow-up.
