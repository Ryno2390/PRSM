RoyaltyDistributor is deployed (canonical is now **v2** `0xfEa9aeB99e02FDb799E2Df3C9195Dc4e5323df7e`; the v1 deploy is retained for legacy claimable only). PRSM is live on Base mainnet — do **not** re-run this runbook as if the contracts are unwired. The procedure below is preserved as a historical record of the T-0 ceremony as planned/executed.

**Audience:** the engineer (you, or a future hand-off) running
`deploy-provenance.js` on Base mainnet to deploy ProvenanceRegistry +
RoyaltyDistributor with the Foundation 2-of-3 Safe as `NETWORK_TREASURY`.

**Companion to:** the operator-side `Multi-Sig_Action_Plan.md` in the
Foundation vault, which covers hardware initialization, Safe deployment,
and signer-key handling. This runbook covers the engineering side: what
the deploy script does, how to dry-run it, how to run it on mainnet, and
how to verify post-deploy state.

**SCOPE.** Task 8 is the immediate next mainnet ceremony after hardware
arrival (ETA 2026-05-01). It deploys 2 contracts:

- **ProvenanceRegistry** — content-provenance accounting for royalty
  routing. No constructor args; non-Ownable.
- **RoyaltyDistributor** — 2% network fee + 8% creator royalty payment
  router. Constructor args `(ftnsToken, registry, networkTreasury)` are
  **immutable**. A typo here is permanent — re-deploy is the only fix.

Task 8 does NOT include:
- The audit-bundle (Phase 3.1 + 7 + 7.1) + Phase 8 emission +
  Phase 7-storage stack — that's a post-external-audit ceremony, weeks-
  to-months later. See `docs/2026-04-30-post-audit-deploy-ceremony-runbook.md`.
- Any modification of the FTNS token at `0x5276a3756C85f2E9e46f6D34386167a209aa16e5`.
  Task 8 just reads the token's `symbol()` to confirm it quacks like FTNS.
- Any role transfer of FTNSToken's DEFAULT_ADMIN_ROLE — that's a
  separate decision on its own timeline.

## 2. Hardhat-local dry-run (T-1d, optional but recommended)

Sanity-checks the deploy script end-to-end without touching mainnet.
Catches: env-var typos, hardhat-config drift, contract-compile errors,
chainId-check edge cases, getter-name regressions.

### 2.1 Single-command dry-run

```bash
./scripts/rehearse-task8.sh
```

That orchestrator (parallels `rehearse-deploy.sh` for the post-audit
ceremony) does:
1. Spins up a hardhat node on 127.0.0.1:8545.
2. Deploys MockERC20 (MFTNS stand-in).
3. Deploys ProvenanceRegistry + RoyaltyDistributor with hardhat
   account #1 as stub treasury.
4. Runs `verify-provenance-deployment.js` against the fresh manifest
   to confirm all immutable getters match.

- Preflight checks pass (FTNS bytecode, MFTNS symbol acceptance,
  chainId log, deployer != treasury).
- ProvenanceRegistry + RoyaltyDistributor both deploy.
- Manifest written to `contracts/deployments/provenance-localhost-*.json`.
- Verifier confirms: `ftns()`, `registry()`, `networkTreasury()`,
  `NETWORK_FEE_BPS()=200` all match.

```bash
  NETWORK_TREASURY=<some-safe-address-on-base> \
```

script with `--network base` against the fork RPC — but that requires
hardhat config tweaking and is overkill for Task 8.

## 3. T-0 mainnet deploy

### 3.1 Operator-side preconditions (re-confirm at T-0)

**Run the pre-deploy checklist:**
```bash
./scripts/pre-task8-checklist.sh
```

This wires findings F1 + F5 + F9 from the Multi-Sig Action Plan
engineering audit into a single executable check. It runs every
verification that `deploy-provenance.js` will do at runtime, BUT
BEFORE the operator burns gas. Verifies (10 numbered checks):

1. All 5 required env vars present (PRIVATE_KEY, FTNS_TOKEN_ADDRESS,
   NETWORK_TREASURY, BASE_RPC_URL, ETHERSCAN_API_KEY).
2. PRIVATE_KEY format (0x + 64 hex chars).
3. Deployer address derived (without echoing the key).
4. BASE_RPC_URL reachable + reports chainId == 8453.
5. ETHERSCAN_API_KEY validates against Etherscan v2 Base endpoint.
6. FTNS_TOKEN_ADDRESS matches canonical 0x5276… (or
   FORCE_NONCANONICAL_FTNS=1).
7. FTNS at the address has bytecode + symbol() == "FTNS".
8. NETWORK_TREASURY != derived deployer address.
9. NETWORK_TREASURY has bytecode (Safe sanity).
10. Deployer balance ≥ 0.003 ETH.

Exits 0 only if all green. **Do not proceed to §3.2 if any check
fails — the deploy script will refuse anyway, and you'll burn fewer
seconds finding out via this checklist than via a half-broadcast
mainnet attempt.**

**Other preconditions:**
- Foundation Safe deployed at `SAFE_ADDR` and verified-owners-correct
  on Basescan.
- Hardware devices not needed for this step — Task 8 is signed by the
  disposable deployer EOA, NOT by Safe signers.

### 3.2 Operator runs the deploy
In the operator's deployer-key terminal (HISTFILE disabled per
Multi-Sig Action Plan Phase 5):

```bash
cd /Users/ryneschultz/Documents/GitHub/PRSM/contracts

export FTNS_TOKEN_ADDRESS="0x5276a3756C85f2E9e46f6D34386167a209aa16e5"
export NETWORK_TREASURY="<SAFE_ADDR>"
export BASE_RPC_URL="<archival RPC URL>"
export PRIVATE_KEY="<DEPLOYER private key, from Phase 5 stash>"
export ETHERSCAN_API_KEY="<Etherscan v2 key>"
export AUTO_VERIFY=1

npx hardhat run scripts/deploy-provenance.js --network base 2>&1 | tee /tmp/mainnet-deploy.log
```

The script will (in order):
1. Validate env vars are set.
2. Checksum FTNS_TOKEN_ADDRESS + NETWORK_TREASURY (refuse zero address).
3. Confirm FTNS bytecode exists + symbol == "FTNS".
4. Log chainId + **hard-fail if `--network=base` and chainId != 8453**.
5. **Hard-fail if FTNS_TOKEN_ADDRESS != canonical 0x5276...** (override
   via `FORCE_NONCANONICAL_FTNS=1` if intentionally deploying against
   a new production token).
6. Refuse if `NETWORK_TREASURY == deployer.address`.
7. **On mainnet: hard-fail if NETWORK_TREASURY is an EOA** (must be
   Safe contract).
8. Deploy ProvenanceRegistry (no constructor args).
9. Deploy RoyaltyDistributor with constructor `(ftns, registry, treasury)`.
10. Write `contracts/deployments/provenance-base-<ts>.json` manifest.
11. If `AUTO_VERIFY=1`: run `hardhat verify` against Basescan for both.

Expected wall-clock: ~30-60 seconds from `npx hardhat run` to manifest
write. The two contract creations are ~15s each on Base; verification
adds ~30s.

### 3.3 Post-deploy hygiene
Per Multi-Sig Action Plan §"Post-deploy hygiene":
- Sweep remaining ETH from deployer back to operator's MetaMask using
  the script in that doc.
- Close the deployer terminal. The disposable key dies with the shell.
- The deployer address now holds no funds, no roles, no privileges —
  even if the key ever leaked it's useless against the deployed
  ProvenanceRegistry / RoyaltyDistributor (non-Ownable, no admin slot,
  no upgrade path; immutable constructor args).

## 4. Post-deploy verification

### 4.1 Manifest sanity check
```bash
cat /Users/ryneschultz/Documents/GitHub/PRSM/contracts/deployments/provenance-base-*.json | tail -1
```

Expected fields:
- `network: "base"`
- `chainId: "8453"`
- `deployer: "<DEPLOYER_ADDR>"`
- `contracts.ProvenanceRegistry: 0x...`
- `contracts.RoyaltyDistributor: 0x...`
- `contracts.FTNSToken: "0x5276a3756C85f2E9e46f6D34386167a209aa16e5"`
- `contracts.NetworkTreasury: "<SAFE_ADDR>"`

### 4.2 On-chain state checks
Easiest path: run the verifier script, which reads the manifest +
calls all immutable getters + asserts equality:

```bash
PROVENANCE_MANIFEST=contracts/deployments/provenance-base-<ts>.json \
  npx hardhat run scripts/verify-provenance-deployment.js --network base
```

It checks: `ftns()`, `registry()`, `networkTreasury()`,
`NETWORK_FEE_BPS()` (must be 200), bytecode at all addresses, FTNS
`symbol()` matches, treasury has bytecode on mainnet (Safe sanity).
Exits 0 if all match the manifest, 1 with explicit `❌` lines if any
don't.

If you'd rather call them by hand:
```bash
DIST=<RoyaltyDistributor address from manifest>
RPC=$BASE_RPC_URL

cast call $DIST "networkTreasury()(address)" --rpc-url $RPC   # expect SAFE_ADDR
cast call $DIST "ftns()(address)" --rpc-url $RPC              # expect 0x5276…
cast call $DIST "registry()(address)" --rpc-url $RPC          # expect ProvenanceRegistry
cast call $DIST "NETWORK_FEE_BPS()(uint16)" --rpc-url $RPC    # expect 200
```

(The actual public state-var name on RoyaltyDistributor is `ftns`, not
`ftnsToken` — `IERC20 public immutable ftns`.)

If any of these don't match, **stop and investigate**. The constructor
args are immutable; a wrong wiring means re-deploy is required.

### 4.3 Basescan verification
If `AUTO_VERIFY=1` was set, both contracts should be marked verified on
basescan.org within ~1 minute of deploy. Confirm by visiting:
- `https://basescan.org/address/<ProvenanceRegistry>`
- `https://basescan.org/address/<RoyaltyDistributor>`

Both should show ✓ "Contract source code verified" with the constructor
args ABI-decoded. If verification didn't auto-succeed (rate-limit, key
issue), retry manually:

```bash
npx hardhat verify --network base <RoyaltyDistributor> \
  "0x5276a3756C85f2E9e46f6D34386167a209aa16e5" \
  "<ProvenanceRegistry>" \
  "<SAFE_ADDR>"
```

## 5. Rollback / abort

The Task 8 deploy is a single-shot deploy of two non-Ownable, non-
upgradeable contracts. There is **no transferOwnership step** to abort
mid-flight — the deploy either completes or it doesn't.

### 5.1 If deploy fails after only ProvenanceRegistry deployed
- Stranded ProvenanceRegistry on mainnet, owned by no one, immutable.
- Re-run the script: it'll deploy a NEW ProvenanceRegistry (different
  address) + a fresh RoyaltyDistributor. The first ProvenanceRegistry is
  orphaned — harmless (no one references it), but consumes ~20¢ of
  burned gas.

### 5.2 If deploy succeeds but constructor args are wrong
- RoyaltyDistributor is **immutable**. Wrong NETWORK_TREASURY (e.g.,
  Safe address typo) means the 2% royalty fee permanently routes to the
  wrong place.
- The fix: re-deploy fresh with corrected args. The wrong-wired
  RoyaltyDistributor is orphaned — operationally harmless (no one
  references it from production code paths) but makes the manifest
  history confusing.
- The mainnet-hardening guards in `deploy-provenance.js` (canonical
  FTNS pin, treasury-is-contract, treasury != deployer) are
  specifically designed to prevent this class of error before broadcast.

### 5.3 If chainId mismatch fires
- `--network=base` but RPC reports a different chainId (e.g., operator's
  BASE_RPC_URL accidentally points at sepolia).
- Script aborts BEFORE any tx is submitted. No mainnet state change.
- Fix the RPC URL and re-run.

```bash
MAINNET_MANIFEST=contracts/deployments/provenance-base-<ts>.json \
  ./scripts/post-task8-handoff-checklist.sh
```

This wires audit finding F3 into executable infrastructure. It:
- Parses the mainnet manifest for the new addresses.
- Greps the repo for every reference to the OLD (sepolia/placeholder)
  addresses — operator walks the list and replaces each with mainnet.
- Surfaces known integration touchpoints (Forta bots, pause-tx
  templates, audit-prep doc, MEMORY.md, .env templates, README,
  CHANGELOG) and flags which exist in the repo.
- Generates a draft PR body at `/tmp/task8-mainnet-handoff-pr-body-<ts>.md`
  ready for `gh pr create --body-file`.
- Generates a project-memory entry stub at
  `/tmp/task8-mainnet-deploy-memory-stub-<ts>.md` that the operator
  reviews + moves into the memory directory.

The script does NOT auto-edit code. Address swaps after a fresh
mainnet deploy deserve human review on each match. The checklist
makes "every place this needs to change" mechanical.

After working through the checklist:
1. Open the address-swap PR using the generated body.
