# v1.6.0 Scope Alignment & Integrity Release — Design Spec

**Status:** Draft pending user review
**Date:** 2026-04-09
**Author:** Brainstorming session (Claude + Ryne)
**Supersedes:** Earlier "AGI framework" framing in `prsm/__init__.py` and README

---

## 1. Goal

Delete all legacy code from PRSM's earlier AGI-framework conception, fix the small set of real bugs that legacy was hiding, and ship a v1.6.0 release whose code matches the current product thesis exactly.

**One-line product thesis:** PRSM is a P2P infrastructure protocol for open-source collaboration. It aggregates latent storage, compute, and data from consumer nodes into a mesh network accessible to third-party LLMs via MCP tools. Contributors earn FTNS tokens for sharing resources.

**This sprint is NOT about new features.** It is about making the codebase truthful: every file in `prsm/` should be in-scope, every docstring should match reality, every test should either pass or point at code that exists.

---

## 2. Context: Why This Sprint Exists

PRSM was previously (over-)conceived as a "decentralized AGI framework" with homegrown AGI orchestration (NWTN), teacher models, distillation pipelines, recursive self-improvement, and coordinated multi-agent workflows. That conception is dead.

A three-pass audit during brainstorming revealed:
1. **First audit** (feature completeness): flagged ~17 skipped tests as "missing subsystems"
2. **Re-audit** (stale skip markers): showed most of those skipped tests were pointing at modules that **actually exist** — the skip markers were stale from earlier development, never cleaned up. The "missing subsystem" panic was largely false.
3. **Scope audit** (deletion manifest): once the user clarified the narrowed scope, it became clear that ~100K LoC of `prsm/` is legacy code that should be deleted, not fixed.

**Key insight:** stale skip markers were silent test debt. Tests appeared to guard code, but never ran. Modules existed in two states simultaneously — "untested but present" and "officially missing." This sprint eliminates both.

**Critical scope preservation:** "NWTN" has two meanings that must not be confused:
- **Old NWTN** (dead): AGI orchestrator, meta-reasoning engine, complete_system, voicebox, agent_forge, hybrid_architecture, chemistry_hybrid_executor, knowledge_graph. **All deleted in this sprint.**
- **Future NWTN** (in-scope): a future fine-tuned LLM for creating WASM agents. **Ring 9 ("The Mind") is its training and deployment infrastructure.** Lives at `prsm/compute/nwtn/training/` and is wired into `prsm/node/node.py:1003`. **Preserved and verified in PR 2.**

1. **Zero legacy code**: every file in `prsm/` is in-scope per the April 2026 scope memo
2. **`prsm/__init__.py` docstring** no longer mentions "AGI", "teacher models", "distillation", "recursive self-improvement" — rewritten to reflect P2P infrastructure + MCP + WASM agents + FTNS

8. **v1.6.0 published to PyPI** and tagged on GitHub
9. **No reduction in functional test coverage** — if a test is deleted, it's because the code it tested was deleted

## 4. Version Strategy

**Single version bump at the end: v1.5.0 → v1.6.0** (one release, not six).

Each of the 6 PRs lands on main in sequence but does **not** bump the version. Only PR 6 bumps the version and publishes. Rationale: users see one clean "scope alignment" release, not six confusing interim versions. Between PRs, the codebase is in a valid but "pre-release" state on main.

**No amended commits after merge. No force-pushes to main. No history rewrites.**

---

## 5. Testing Strategy

1. **Every PR runs the full test suite** before merge. No skipping tests to land faster.
2. **Deleted tests are deleted with their code.** If a test is pointed at `prsm.compute.distillation`, it gets deleted in the same PR as `prsm/compute/distillation/`. No orphaned tests.
3. **Stale skip removal is paired with fix-or-delete.** For each stale skip marker: remove the skip, run the test. If it passes → done. If it fails and the tested code is legacy → delete the test. If it fails and the tested code is in-scope → fix the real bug.
4. **Ring 9 gets extra validation in PR 2**: run `tests/integration/test_ring9_mind.py` and `tests/e2e/test_ring_cross_node.py::test_ring9_*` before AND after the NWTN trim. Both must pass with identical behavior.
5. **No skipping pre-commit hooks** — ever.
6. **PyPI release only after all 6 PRs merge and CI green on main.**

---

## 6. Rollback Strategy

Each PR is a conventional merge commit on main with a clear scope. If any PR introduces a regression not caught until after a later PR merges, we `git revert` that specific PR — the other phases stay. Each PR is small enough to revert cleanly.

If a regression straddles PRs (e.g., a bug only manifests after PR 3 *and* PR 4 both land), we revert forward from the earliest PR that caused it. The sprint restarts from a working main.
