## 1. Engagement summary (TL;DR)

PRSM operates a two-entity structure:

- **PRSM Foundation** — Cayman Islands nonprofit, public-good
  steward of the protocol + FTNS token, 2-of-3 multisig governance.
- **Prismatica** — Delaware C-corp, for-profit subsidiary, Reg D
  506(c) equity raise to fund development.

We are seeking counsel in **two jurisdictions**:

- **Track A — Cayman counsel** for Foundation governance (entity
  formation review, charter amendments, councilor compensation,
  treasury policy, FATF/OFAC compliance).
- **Track B — US securities counsel** for (1) Prismatica's Reg D
  506(c) equity raise, (2) FTNS classification opinion (Howey
  analysis: utility vs. security), and (3) KYC/AML
  obligations across both entities.

The two counsel must be able to work in parallel and refer to each
other's work.

## 2. Why us / why now

PRSM is pre-mainnet at the time of this RFP. The audit-bundle
contracts are in pre-deploy review (L3 + L4 RFPs out as of
2026-05-05). Phase 1.3 contracts (FTNS token + ProvenanceRegistry +
RoyaltyDistributor) are deployed to Base mainnet but not yet at
public-launch volume.

Counsel work is gating for:

  investor outreach.

  before any non-airdrop public distribution path (DEX listing, CEX
  listing, public sale).
- **KYC vendor selection** — Phase 5 §8.1 memo selected a
  KYC vendor pending counsel sign-off on the chosen workflow.

We have done the internal scoping work to make counsel engagement
efficient (see §6 pre-engagement artifacts).

## 4. Scope of work

### 4.1 Track A — Cayman counsel

**Foundation entity review:**

- [ ] Review existing Foundation formation documents (charter,
      bylaws, council appointments)
- [ ] Validate that Foundation structure complies with Cayman
      Foundation Companies Act 2017
- [ ] Identify any required charter amendments before public launch
- [ ] Council compensation policy review
- [ ] FATF / OFAC sanctions screening compliance review
- [ ] Treasury policy review (per L11 council ratification —
      pending — including risk-transfer / insurance approach)

**Operational governance:**

- [ ] 2-of-3 multisig formal recognition (signing authority,
      delegation, revocation)
- [ ] Future DAO transition pathway (per Q6 ratification — Cayman
      regulatory treatment of DAO-style governance)
- [ ] Quarterly council meeting + voting protocol
- [ ] Officer succession (founder bus-factor mitigation per task
      #134; deputy founder in `docs/security/D7_DEPUTY_FOUNDER_SUCCESSION.md`)

**Cross-jurisdictional coordination:**

- [ ] Boundary subsidiary policy (per
      `docs/2026-04-23-prsm-policy-jurisdiction-1.md`) — review with
      US counsel for tax + securities implications
- [ ] Information-sharing arrangement between Foundation and
      Prismatica
- [ ] Conflict-of-interest framework (officer-overlap between
      entities)

### 4.2 Track B — US securities counsel

**Prismatica Reg D 506(c) equity raise:**

- [ ] Review Prismatica formation (Delaware C-corp)
- [ ] Draft / review Private Placement Memorandum (PPM)
- [ ] Cap table review + accredited-investor verification process
- [ ] Reg D filing (Form D)
- [ ] Bad-actor disqualification check (Rule 506(d))
- [ ] State-level blue-sky filings for materially-marketed states

**FTNS classification opinion:**

- [ ] Howey analysis: investment of money, common enterprise,
      expectation of profit, derived from efforts of others
- [ ] Memorandum on whether FTNS is a security under US securities
      laws
- [ ] Identify any structural changes needed to support a
      "utility token" position (if applicable)
- [ ] Coordinate with Cayman counsel on cross-jurisdictional
      classification

**KYC/AML compliance:**

- [ ] Review Phase 5 KYC vendor selection
      (`docs/2026-04-27-phase5-kyc-vendor-decision.md`) and chosen
      workflow
- [ ] FATF Travel Rule applicability + implementation guidance
- [ ] OFAC sanctions screening (technical implementation review)
- [ ] CCPA / state-level privacy compliance for KYC data handling
- [ ] Sub-licensee terms (if KYC vendor is on-platform)

**Public-distribution paths:**

- [ ] If FTNS is utility-token positioned: review DEX listing, CEX
      listing, distribution mechanics
- [ ] If FTNS is security positioned: Reg A+ / Reg S / Reg CF
      pathway analysis
- [ ] Tax characterization (capital gain vs ordinary income) for
      protocol participants

**Cross-jurisdictional coordination:**

- [ ] Foundation/Prismatica relationship review (per
      jurisdiction policy)
- [ ] Prismatica's licensing of code to Foundation (or vice versa)
- [ ] IP transfers between entities — tax treatment

### 4.3 Out of scope

| Topic | Why out of scope |

| **L8 (this RFP)** | **Legal / regulatory** | **Two counsel — Cayman + US** |

L8 covers the *legal* threat surface, which is distinct from all
technical surfaces. A legal opinion that's wrong creates SEC
enforcement risk; a code bug creates fund-loss risk; both are
"audit findings" in the layered defense-in-depth model.
