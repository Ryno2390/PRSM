## 1. Context & Goals

Phases 3.x.2 + 3.x.4 ship signed artifacts (model manifests + privacy-budget entries) whose verification depends on a local `node.pubkey` sidecar at the journal/registry root. Both tags explicitly flag this as a **filesystem trust boundary**: an attacker with write access to the root can replace the sidecar AND re-sign every artifact under their own key — and the local verifier accepts the substitute.

The fix is to move the publisher's public key out of "trust the local file" and into "trust the on-chain registry." A verifier then:

1. Reads `publisher_node_id` from the signed artifact (manifest, budget entry, receipt).
2. Looks up `publisher_node_id → public_key` on-chain.
3. Calls the existing `verify_manifest / verify_entry / verify_receipt` with the anchored key.

The trust anchor becomes the contract's admin (Foundation multisig — same governance posture as FTNS), not the local filesystem.

This phase intentionally does NOT cover **DHT distribution** (the second half of the original 3.x.3 placeholder). Trust is the active blocker; availability is not. DHT is deferred to a future Phase 3.x.5.

### 1.1 Why now

- Phases 3.x.2 + 3.x.4 just shipped with documented trust caveats. Closing them in the same week the persistence shipped keeps the "every signed artifact is cross-node verifiable" narrative coherent for any partner doing a deep review.
- The Solidity verifier infrastructure (Phase 1.3 production Ed25519 verifier) is warm and the contract-deploy pipeline is exercised.
- Sepolia FTNS testnet already exists; the anchor contract deploys alongside.

### 1.2 Non-goals for Phase 3.x.3

- **Not DHT distribution.** Replication of manifest/journal bytes across operators is a separate concern (availability vs. trust). Deferred to Phase 3.x.5+.
- **Not artifact-CID anchors.** v1 anchors only `publisher_node_id → public_key`. Per-artifact CID anchors (proving "this specific manifest exists on-chain") add cost per write that doesn't pay for itself until DHT lands.
- **Not key rotation.** Write-once-per-`node_id` with a multisig override for emergencies. Versioned rotation history is a v2 feature once a real rotation use case exists.
- **Not Base mainnet deploy.** Sepolia only for v1. Base bundles into the existing Phase 1.3 audit clock.
- **Not a hard-flip of existing verifiers.** `verify_manifest / verify_entry / verify_receipt` continue to accept `public_key_b64` directly (offline-verifier path). The anchor is an additive resolver layer.
- **Not a self-improving / governance-gated registry.** Permissionless registration with cryptographic proof of ownership (sha256-derived nodeId verification on-chain). Foundation multisig only intervenes via the admin override.

### 1.3 Backwards compatibility

- All existing call sites of `verify_manifest / verify_entry / verify_receipt` with `public_key_b64` keep working unchanged.
- Existing `FilesystemModelRegistry` and `FilesystemPrivacyBudgetStore` constructors are not modified — anchor support is added as an optional `anchor=` kwarg.
- A node with no anchor configured continues to use sidecar-only trust (Phase 3.x.2 / 3.x.4 behavior).
- The contract has no fee. Registration costs a one-time tx fee (~50k gas).

### 1.4 Trust model upgrade

| Concern | Phase 3.x.2 / 3.x.4 defense | Phase 3.x.3 upgrade |
|---|---|---|
| Operator swaps `node.pubkey` sidecar AND re-signs all artifacts | NONE — verifier accepts substitute | Anchor lookup overrides sidecar; substitute fails verify |
| Operator deletes journal entirely (forces fresh sequence 0) | Documented limitation | Same — anchor doesn't address availability |
| Cross-node verifier needs publisher's key | Trust local copy / out-of-band exchange | Anchor lookup |
| Adversary registers victim's `node_id` on-chain | N/A | Contract derives `node_id` from `sha256(public_key)` and rejects mismatched registrations — adversary can only register their own keys |
| Compromised publisher key needs revocation | N/A | Multisig admin override (`adminOverride(node_id, new_key)`) — emergency-only path |

## 3. Architecture

```
┌─────────────────────────────────────────────────────────────────────┐
│ Verifier (any client holding only a signed artifact)               │
│   verify_manifest_with_anchor(manifest, anchor_client) → bool       │
│         │                                                            │
│         ▼                                                            │
│   anchor_client.lookup(manifest.publisher_node_id)                  │
│         │  → public_key_b64                                         │
│         ▼                                                            │
│   verify_manifest(manifest, public_key_b64=resolved)                │
└─────────────────────────────────────────────────────────────────────┘
                                  │
                                  │ web3 read call (free)
                                  ▼
┌─────────────────────────────────────────────────────────────────────┐
│ PublisherKeyAnchor.sol on Sepolia (later: Base mainnet)             │
│   mapping(bytes16 => bytes) publisherKeys                           │
│     ↑ key = first 16 bytes of sha256(pubkey)                        │
│     ↑ value = raw 32-byte Ed25519 pubkey                            │
│   register(bytes pubkey) — derives node_id on-chain; write-once     │
│   adminOverride(bytes16 nodeId, bytes newKey) — multisig only       │
└─────────────────────────────────────────────────────────────────────┘
                                  ▲
                                  │ tx (~50k gas, paid by publisher)
                                  │
┌─────────────────────────────────────────────────────────────────────┐
│ Publisher node (on first deploy)                                    │
│   anchor_client.register_self(identity)                             │
│     → wallet signs + broadcasts register(identity.public_key_bytes) │
└─────────────────────────────────────────────────────────────────────┘
```

### 3.1 Solidity contract format

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

contract PublisherKeyAnchor {
    address public immutable admin;
    mapping(bytes16 => bytes) public publisherKeys;
    mapping(bytes16 => uint64) public registeredAt;

    event PublisherRegistered(
        bytes16 indexed nodeId,
        bytes publicKey,
        uint64 timestamp
    );
    event PublisherOverridden(
        bytes16 indexed nodeId,
        bytes oldKey,
        bytes newKey,
        uint64 timestamp
    );

    error AlreadyRegistered();
    error InvalidPublicKeyLength();
    error NotAdmin();
    error NotRegistered();

    constructor(address _admin) {
        require(_admin != address(0), "admin=zero");
        admin = _admin;
    }

    /// Register a publisher's public key. Anyone can call; the contract
    /// derives node_id from sha256(publicKey) on-chain so the caller
    /// cannot forge a registration for someone else's key. Write-once
    /// per node_id; admin override is the only way to change.
    function register(bytes calldata publicKey) external {
        if (publicKey.length != 32) revert InvalidPublicKeyLength();
        bytes16 nodeId = bytes16(sha256(publicKey));
        if (publisherKeys[nodeId].length != 0) revert AlreadyRegistered();
        publisherKeys[nodeId] = publicKey;
        registeredAt[nodeId] = uint64(block.timestamp);
        emit PublisherRegistered(nodeId, publicKey, uint64(block.timestamp));
    }

    /// Emergency-only path. Multisig admin overrides a stored key
    /// (e.g., compromised publisher). Bypasses write-once.
    function adminOverride(bytes16 nodeId, bytes calldata newKey) external {
        if (msg.sender != admin) revert NotAdmin();
        if (publisherKeys[nodeId].length == 0) revert NotRegistered();
        if (newKey.length != 32) revert InvalidPublicKeyLength();
        bytes memory oldKey = publisherKeys[nodeId];
        publisherKeys[nodeId] = newKey;
        registeredAt[nodeId] = uint64(block.timestamp);
        emit PublisherOverridden(nodeId, oldKey, newKey, uint64(block.timestamp));
    }

    /// View. Returns empty bytes if not registered.
    function lookup(bytes16 nodeId) external view returns (bytes memory) {
        return publisherKeys[nodeId];
    }
}
```

**Why `bytes16` instead of `string`**: Solidity strings are awkward + expensive. `bytes16` is the raw first 16 bytes of `sha256(pubkey)` — the same width PRSM's `NodeIdentity` truncates to and then hex-encodes for human display. Verifiers convert the artifact's hex `node_id` to `bytes16` client-side before calling `lookup`.

**Why 32-byte raw pubkey instead of base64 string**: storage efficiency (32 vs ~44 bytes per publisher × N publishers). Client base64-encodes after lookup for the existing `verify_X(public_key_b64=...)` API.

### 3.2 Python client contract

```python
class PublisherKeyAnchorClient:
    """Wraps the on-chain PublisherKeyAnchor contract.

    Lookup is the hot path — verifiers call it on every cross-node
    artifact verification. Cached in-memory with a configurable TTL
    (default 1 hour) so repeated verification of artifacts from the
    same publisher doesn't hit the chain.
    """

    def __init__(
        self,
        contract_address: str,
        web3_provider: str,
        *,
        cache_ttl_seconds: int = 3600,
        wallet_private_key: Optional[str] = None,
    ): ...

    def register_self(self, identity: NodeIdentity) -> str:
        """Register THIS node's pubkey on-chain. Returns tx hash.

        Requires wallet_private_key configured at construction (registration
        costs gas; the wallet must hold ETH). Idempotent at the
        contract level: if already registered, the contract reverts and
        the client surfaces the revert as PublisherAlreadyRegisteredError.
        """

    def lookup(self, node_id: str) -> Optional[str]:
        """Look up a publisher's pubkey by node_id (hex string).

        Returns base64-encoded public key, or None if not registered.
        Hits the cache first; falls through to a contract read on miss.
        """
```

### 3.3 Verifier wrapper contract

Each artifact type gets a 3-line wrapper that does anchor lookup + delegates:

```python
def verify_manifest_with_anchor(
    manifest: ModelManifest,
    anchor: PublisherKeyAnchorClient,
) -> bool:
    pubkey = anchor.lookup(manifest.publisher_node_id)
    if pubkey is None:
        return False
    return verify_manifest(manifest, public_key_b64=pubkey)
```

Mirrors for `verify_entry_with_anchor` (PrivacyBudgetEntry) and `verify_receipt_with_anchor` (InferenceReceipt → uses `settler_node_id`).

## 6. Risk register

| Risk | Likelihood | Impact | Mitigation |
|---|---|---|---|
| Adversary races to register a victim's `node_id` | Low | High | Contract derives `node_id` from `sha256(pubkey)` on-chain; adversary can only register their own pubkeys. Tested. |
| Compromised publisher key needs revocation | Low | High | `adminOverride` (multisig) is the v1 escape hatch. Documented as emergency-only. |
| Sepolia RPC outage breaks verifier | Medium | Medium | Anchor client cache (1-hour TTL) absorbs short outages. Documented. Verifiers must surface anchor failure as "verification failed," NOT silently fall back to sidecar (would defeat the trust upgrade). |
| Multisig admin key compromised | Low | Critical | Same trust assumption as FTNS multisig — Foundation governance. Out of scope for this phase. |
| Per-publisher gas cost is a barrier to adoption | Low | Low | ~50k gas at Sepolia testnet rates is trivial. On Base mainnet at $5/M gas, that's <$0.01 per publisher. Documented. |
| Cache poisoning via stale lookup after an admin override | Low | Medium | TTL bounds staleness to 1 hour; explicit `client.invalidate(node_id)` available for operators who know an override just happened. |
| Solidity verifier bug (sha256 derivation) lets a forged registration through | Low | Critical | Hardhat tests cover the derivation explicitly + property-based fuzz with random pubkeys. Audit gates Base deploy. |
| Verifier wrappers swallow an anchor RPC error as `False` instead of distinguishing | Medium | Medium | Wrappers return `False` on lookup failure (same fail-closed contract as `verify_manifest`). Operators get the underlying error via `client.last_error` when they need to distinguish. Documented. |
